# Регламент создания объекта в AXENTA

## Способы создания объекта

Объект можно создать двумя способами:

**Способ 1** — из меню CMS в разделе **«Объекты»**, нажав значок **«+»**:

![Список объектов в CMS](https://raw.githubusercontent.com/KeeGooRoomiE/wikinest_avtoscan/HEAD/docs/assets/images/axenta-obj/ao-p1-objects-list.png)

**Способ 2** — войти под административным пользователем через **«Войти в мониторинг»** и создать объект из раздела **«Объекты»** → **«+»**:

![Административный пользователь](https://raw.githubusercontent.com/KeeGooRoomiE/wikinest_avtoscan/HEAD/docs/assets/images/axenta-obj/ao-p2-admin-user.png)

![Раздел объектов](https://raw.githubusercontent.com/KeeGooRoomiE/wikinest_avtoscan/HEAD/docs/assets/images/axenta-obj/ao-p2-objects-section.png)

---

## Заполнение формы создания объекта

В обоих случаях откроется окно **«Создание объекта»**. Заполнить:
- **Имя** — название ТС
- **Тип устройства** — выбрать модель навигационного терминала
- **Уникальный ID** — IMEI терминала
- **Телефон 1** — номер SIM-карты, установленной в терминале

> Если тип ТС неизвестен заранее — по умолчанию выбрать иконку **«красной машины»**.

![Форма создания объекта](https://raw.githubusercontent.com/KeeGooRoomiE/wikinest_avtoscan/HEAD/docs/assets/images/axenta-obj/ao-p3-create-form.png)

## Настройка объекта

### 1. Вкладка «Дополнительно»

Выставить параметры как на скриншотах:
- **Предел скорости в городе**: `79` км/ч
- **Максимальный интервал между сообщениями**: `800` сек
- **Коэффициент пробега**: `1`
- **Цветовой режим трека**: «По датчику» → **Зажигание**

![Дополнительные параметры](https://raw.githubusercontent.com/KeeGooRoomiE/wikinest_avtoscan/HEAD/docs/assets/images/axenta-obj/ao-p4-additional-settings.png)

![Цветовой режим трека](https://raw.githubusercontent.com/KeeGooRoomiE/wikinest_avtoscan/HEAD/docs/assets/images/axenta-obj/ao-p4-track-color.png)

### 2. Вкладка «Датчики»

Порядок датчиков **строго фиксирован**:

```
Зажигание → Напряжение → CAN-датчики → Датчики доп. оборудования → Датчики топлива
```

![Список датчиков](https://raw.githubusercontent.com/KeeGooRoomiE/wikinest_avtoscan/HEAD/docs/assets/images/axenta-obj/ao-p5-sensors-list.png)

#### Рекомендации по настройке датчика уровня топлива

![Свойства датчика топлива](https://raw.githubusercontent.com/KeeGooRoomiE/wikinest_avtoscan/HEAD/docs/assets/images/axenta-obj/ao-p6-fuel-sensor-settings.png)

**Медианная фильтрация** — убирает «шум» от вибрации, уклона дороги и волн в баке:

| Тип ТС | Значение | Пояснение |
|---|---|---|
| Легковые (ровные дороги) | 2–3 | Малый шум, высокая точность |
| Грузовые (трассы, вибрация) | 4–6 | Стандартное сглаживание |
| Спецтехника (бездорожье, карьеры) | 8–10+ | Экстремальная тряска; реакция на слив/заправку замедляется на 1–2 мин |

**Настройки слива:**

![Настройки слива](https://raw.githubusercontent.com/KeeGooRoomiE/wikinest_avtoscan/HEAD/docs/assets/images/axenta-obj/ao-p7-drain-settings.png)

**Минимальный объём заправки** — порог, ниже которого колебание уровня не считается заправкой:

| Тип ТС | Значение |
|---|---|
| Легковые | 5–7 л |
| Грузовые | 12–20 л |
| Агрессивная среда | 20–30 л (или 3–5% объёма бака) |

**Минимальный объём слива** — порог, ниже которого расход не считается сливом:

| Тип ТС | Значение |
|---|---|
| Легковые | 5–7 л |
| Грузовые | 10–15 л |
| Агрессивная среда | 20–25 л |

> ⚠️ Значение меньше 3 литров для легковых даёт множество ложных тревог.

### 3. Вкладки «Характеристики» и «Настраиваемые поля»

Оставить **пустыми**.

### 4. Вкладка «Детектор поездок»

Установить **«Датчик зажигания»** как метод определения движения:

| Параметр | Значение |
|---|---|
| Определение движения | Датчик зажигания |
| Минимальная скорость | 1 км/ч |
| Минимальное время стоянки | 300 с |
| Минимальное время поездки | 60 с |
| Минимальная дистанция поездки | 100 м |
| Максимальное расстояние между сообщениями | 10 000 м |

![Детектор поездок](https://raw.githubusercontent.com/KeeGooRoomiE/wikinest_avtoscan/HEAD/docs/assets/images/axenta-obj/ao-p9-trip-detector.png)

### 5. Вкладки «Команды» и «Техобслуживание»

Заполнять **по потребностям клиента**.

### 6. Доступ

Выдать доступ на объект пользователю клиента (см. [регламент создания учётной записи](add-car-reglament)).
