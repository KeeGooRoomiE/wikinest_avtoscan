# Folder structure

Pages in WikiNest map directly to files in the `docs/` directory. The sidebar tree mirrors the directory structure.

## Basic structure

```
docs/
├── index.md              → home page (/)
├── getting-started.md    → /getting-started
├── setup/
│   ├── _meta.json        → folder display name
│   ├── installation.md   → /setup/installation
│   └── configuration.md  → /setup/configuration
└── ops/
    ├── _meta.json
    ├── monitoring.md     → /ops/monitoring
    └── servers/
        ├── _meta.json
        └── nginx.md      → /ops/servers/nginx
```

## Depth

There is no limit on folder nesting depth. The sidebar renders each level with increasing indentation.

## Naming conventions

Use lowercase letters, numbers, and hyphens for file and folder names. Avoid spaces, uppercase letters, and non-ASCII characters in paths.

| Good | Avoid |
|---|---|
| `getting-started.md` | `Getting Started.md` |
| `server-config.md` | `server config.md` |
| `ops/monitoring.md` | `ops/Мониторинг.md` |

Page titles displayed in the sidebar come from the first `# Heading` in the file — write those in any language.

## _meta.json

Place `_meta.json` inside any folder to set a custom display name:

```json
{ "title": "Operations", "order": 1 }
```

Without `_meta.json`, the folder name is used with hyphens and underscores replaced by spaces and the first letter capitalized.

## tree.json

`tree.json` is auto-generated by GitHub Actions and should not be edited manually. It contains the flat list of all pages with their paths, titles, and excerpts used by the sidebar and search.

Example entry:

```json
{
  "path": "setup/installation",
  "title": "Installation",
  "excerpt": "WikiNest requires no build step...",
  "parts": ["setup", "installation"],
  "folder_titles": {
    "setup": "Setup & Installation"
  }
}
```
