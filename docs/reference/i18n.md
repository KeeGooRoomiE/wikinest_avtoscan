# Internationalization

WikiNest supports multiple UI languages. Page content language is not managed — write pages in any language you want.

## How it works

At startup, WikiNest fetches `i18n/<lang>.json` and uses its values for all UI labels. The language is chosen by:

1. The value stored in `localStorage` (user's last selection)
2. The `lang` field in `config.json` (site default)
3. Fallback to `en`

If a key is missing from the loaded language file, the key name itself is displayed as fallback.

## Switching languages

Use the language dropdown in the header. The selection is saved per-browser in `localStorage`.

## Adding a language

1. Copy `i18n/en.json` to `i18n/<lang>.json` using the ISO 639-1 language code (e.g. `de`, `fr`, `ja`)
2. Translate all values — keep all keys unchanged
3. Add an option to the `#lang-switcher` select in `index.html`:

```html
<select id="lang-switcher" onchange="switchLang(this.value)">
  <option value="en">EN</option>
  <option value="ru">RU</option>
  <option value="de">DE</option>  <!-- add this -->
</select>
```

4. Commit and push
