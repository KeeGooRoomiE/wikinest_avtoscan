# WikiNest

**Git-native wiki for teams. Edit markdown in the browser, save as commits. No server, no build step, no dependencies beyond GitHub.**

## What it is

WikiNest turns a GitHub repository into a fully functional wiki — with a visual editor, sidebar navigation, full-text search, and password-protected editing — without requiring a server, a database, or a build pipeline.

Every page is a markdown file in `docs/`. Every edit is a commit. The sidebar tree is auto-generated by GitHub Actions. The entire frontend is a single `index.html` + `style.css`.

## How it works

```
Browser
  │
  ├─ reads  ── GitHub raw content API ── tree.json, .md files
  └─ writes ── GitHub REST API PUT /repos/.../contents/...
                        │
                        └── commit to repo
                                │
                                └── GitHub Actions
                                        ├── rebuilds tree.json
                                        └── deploys to GitHub Pages
```

No backend. No database. Your PAT is encrypted with your password and stored in `localStorage` — it only ever reaches `api.github.com`.

---

## Features

- **In-browser markdown editor** — split-view: raw markdown on the left, live preview on the right
- **Tree navigation** — sidebar mirrors the `docs/` folder structure, unlimited nesting
- **Folder aliases** — custom display names via `_meta.json` without renaming directories
- **Full-text search** — client-side, ⌘K / Ctrl+K, searches titles and excerpts
- **Password-protected editing** — SHA-256 hash in `config.json`, PAT encrypted in localStorage
- **Multi-maintainer support** — `config.json` is the source of truth; each editor brings their own PAT
- **i18n** — UI strings in `i18n/<lang>.json`; English and Russian included
- **Dark mode** — follows `prefers-color-scheme` automatically
- **Mobile-friendly** — sidebar collapses to hamburger, content adapts
- **Zero dependencies** — one CDN script (`marked.js`), vanilla JS, no framework, no npm

## Quick start

### 1. Fork and enable Pages

Fork this repository. Go to **Settings → Pages → Source** and select **GitHub Actions**. Go to **Settings → Actions → General → Workflow permissions** and select **Read and write permissions**.

### 2. Add owner and repo to config.json

Edit `config.json` in the repository root:

```json
{
  "site_name": "My Wiki",
  "lang": "en",
  "owner": "your-github-username",
  "repo": "wikinest",
  "edit_password_hash": ""
}
```

Commit and push. GitHub Actions will build and deploy automatically (~1 min).

### 3. Create a Personal Access Token

**GitHub → Settings → Developer settings → Personal access tokens (classic) → Generate new token**

Required scope: `repo` ✓

### 4. Configure the wiki

Open your deployed site. Click the **⚙** icon. Enter your PAT and an edit password. Click Save.

That's it. The wiki is ready.

## Configuration

`config.json` is committed to the repository and is public. Never put secrets here.

| Field | Type | Description |
|---|---|---|
| `site_name` | string | Displayed in the header and browser tab |
| `lang` | string | Default UI language (`en`, `ru`) |
| `owner` | string | GitHub username or org that owns the repository |
| `repo` | string | Repository name |
| `edit_password_hash` | string | SHA-256 hash of the edit password. Set via the ⚙ UI. |

`config.json` is loaded fresh on every page open — changes are picked up by all users without any local action.

## Authentication and security model

### Roles

| Role | What they need |
|---|---|
| **Reader** | Just open the site. No setup required. |
| **Editor** | Own PAT with `repo` scope + know the edit password |
| **Maintainer** | Write access to the repository (to change `config.json`, add pages via git) |

### How the edit password works

1. You enter a password in the ⚙ settings
2. SHA-256 hash is computed in the browser and stored in `config.json` via a commit
3. Your PAT is XOR-encrypted with that password and stored in `localStorage`
4. On each session, you enter the password → PAT is decrypted in memory → editing is available

### What is and isn't protected

| Protected | Not protected |
|---|---|
| PAT (encrypted in localStorage) | Page content (public GitHub repo) |
| Edit UI (password required) | `config.json` (public, readable by anyone) |
| Commits (require valid PAT) | Password hash (public, but SHA-256 is one-way) |

> WikiNest is designed for **internal or semi-public documentation** where content is not secret but editing should be controlled. Do not use it for confidential content in a public repository.

### Generating the hash manually

```js
// Run in browser console on any page
const pw = 'your-password';
const buf = await crypto.subtle.digest('SHA-256', new TextEncoder().encode(pw));
console.log(Array.from(new Uint8Array(buf)).map(b => b.toString(16).padStart(2,'0')).join(''));
```

## Creating pages

Click **+ New page** in the sidebar. Enter a path:

| Input | Creates |
|---|---|
| `getting-started` | `docs/getting-started.md` |
| `ops/monitoring` | `docs/ops/monitoring.md` |
| `ops/servers/nginx` | `docs/ops/servers/nginx.md` |

Folders are created automatically. Page titles come from the first `# Heading` in the file.

---

## Folder aliases

Add `_meta.json` inside any folder to set a display name:

```json
{ "title": "Operations", "order": 1 }
```

`_meta.json` files are ignored by the page tree — they never appear as pages in the sidebar.

---

## Internationalization

Add a language by:

1. Copying `i18n/en.json` to `i18n/<lang>.json` and translating all values
2. Adding `<option value="<lang>">XX</option>` to `#lang-switcher` in `index.html`

All keys must stay in English. Missing keys fall back to the key name.

## GitHub Actions

Two workflows are included:

**`deploy.yml`** — triggers on every push to `main`. Rebuilds `tree.json`, commits it if changed, then deploys to GitHub Pages.

**`build-tree.yml`** — triggers only on changes to `docs/` or `config.json`. Handles the case where WikiNest UI commits directly to `docs/` — without re-triggering the full deploy.

Required permission: **Settings → Actions → General → Workflow permissions → Read and write permissions**

## Customization

All colors are CSS variables in `style.css`:

```css
:root {
  --accent:        #185FA5;   /* primary blue */
  --accent2:       #0c447c;   /* hover state */
  --accent-bg:     #E6F1FB;   /* accent tint backgrounds */
  --accent-border: #B5D4F4;   /* accent borders */
  --sidebar-w:     240px;
  --toc-w:         160px;
  --header-h:      48px;
}
```

Dark mode variables are in the `@media (prefers-color-scheme: dark)` block in the same file.

## Known limitations

- **Search is excerpt-only** — indexes only the first ~140 characters of each page, not full content. Increase `[:140]` in the workflow scripts to index more, at the cost of a larger `tree.json`.
- **No image upload UI** — images must be committed to the repository manually and referenced by path.
- **Single branch** — WikiNest always reads from and writes to the default branch (`main`). Branch-based workflows are not supported.
- **No conflict resolution** — if two people edit the same page simultaneously, the second save will fail with a SHA mismatch error from the GitHub API.
- **PAT rotation** — when a PAT expires, each user must re-enter their token in the ⚙ settings.
- **No offline support** — requires a GitHub API connection for reading and writing.

## Contributing

PRs are welcome. For large changes, open an issue first to discuss.

### Setup

No build step required. Open `index.html` directly in a browser for local development. For GitHub API calls to work locally, configure your own fork in the ⚙ settings.

### Conventions

- Commits: `type: description` — `feat:`, `fix:`, `docs:`, `chore:`
- Keep `index.html` and `style.css` free of external dependencies beyond the CDN allowlist
- i18n: all new UI strings must have entries in both `en.json` and `ru.json`
- Test dark mode before submitting UI changes

### What to contribute

- Translations (`i18n/<lang>.json` + option in `index.html`)
- Bug fixes
- Accessibility improvements
- Documentation

## Troubleshooting

**Sidebar empty after creating/editing a page**
GitHub Actions needs ~30–60s to rebuild `tree.json`. Reload after that.

**"Could not load — check settings"**
Verify `owner` and `repo` in `config.json` are correct. Check that GitHub Pages is enabled and the first deploy completed successfully.

**Actions fail with "Permission denied"**
Settings → Actions → General → Workflow permissions → **Read and write permissions**.

**Password doesn't work after manually editing config.json**
Use the browser console method above to generate the exact SHA-256 hash.

**Save fails with "SHA mismatch"**
Someone else edited the same page. Reload, copy your changes, and re-apply them.

## Working with AI tools

Feed `docs/reference/` pages to any AI tool (Claude, ChatGPT, Cursor) before making changes.

Key files:
- `docs/reference/folder-structure.md` — how the page tree works, `tree.json` schema
- `docs/reference/i18n.md` — full list of UI string keys
- `docs/reference/markdown.md` — what markdown is supported

When asking an AI to add a feature to `index.html`, always include the current file — there is no build system, so the full source is the context.
